# SmartHIS 技术栈架构与调用方法

## 1. 项目定位

SmartHIS 是一个面向首视 Smart 系列软件的医院业务与接口仿真平台，用于数字化手术室、手术示教、远程会诊、手术室门口屏、中央监控、排队叫号、厂商接口联调和演示测试。

当前项目不是生产级 HIS，也不是传统前后端分离系统。它是一个单进程 Node.js HTTP 服务，使用内存状态模拟医院业务数据，并提供浏览器页面、REST API、FHIR、DICOMweb、HL7、HIS 兼容接口和厂商只读数据库快照。

## 2. 技术栈

| 层级 | 技术/实现 | 说明 |
|---|---|---|
| 运行时 | Node.js >= 20 | 使用 ES Module，`type: module` |
| HTTP 服务 | Node 原生 `http` | 未引入 Express/Koa 等框架 |
| 页面渲染 | 原生 HTML/CSS/JavaScript 字符串渲染 | 后端直接返回控制台、大屏、终端模拟页面 |
| 数据状态 | 内存对象 | 启动时由种子数据生成，重启后恢复初始状态 |
| 厂商快照 | Node 内置 `node:sqlite` + SQLite 文件 | 生成 `data/smarthis-vendor-readonly.sqlite` |
| 标准接口 | REST、FHIR R4 风格、DICOMweb、HL7 v2 文本 | 用于联调和演示，不是完整生产协议栈 |
| 测试 | Node 原生 `node:test` | 通过 `npm test` 执行 |

## 5. 启动方法

在项目根目录执行：

```powershell
cd D:\我的工作\AOV\SmartHIS
npm start
```

默认监听：

```text
http://127.0.0.1:7070
```

如果真实安卓终端或同局域网设备需要访问，应监听所有网卡：

```powershell
$env:HOST="0.0.0.0"
$env:PORT="7070"
npm start
```

如需换端口：

```powershell
$env:PORT="7080"
npm start
```

## 6. 环境变量

| 变量 | 默认值 | 作用 |
|---|---:|---|
| `HOST` | `127.0.0.1` | HTTP 监听地址 |
| `PORT` | `7070` | HTTP 监听端口 |
| `SMARTHIS_NATURAL_OPERATION` | 启用 | 设置为 `0` 可关闭自然运转 |
| `SMARTHIS_NATURAL_PATIENT_COUNT` | `50` | 初始模拟病人数 |
| `SMARTHIS_NATURAL_ROOM_COUNT` | `6` | 开放手术间数量 |
| `SMARTHIS_NATURAL_DAILY_ADMISSIONS` | `6` | 每日计划新入院人数 |
| `SMARTHIS_NATURAL_TICK_MS` | `30000` | 自动推进间隔，单位毫秒 |
| `SMARTHIS_VENDOR_DB_PATH` | `data/smarthis-vendor-readonly.sqlite` | 厂商 SQLite 快照路径 |
| `SMARTHIS_VENDOR_DATA_SCOPE` | `display` | 厂商快照字段范围，支持 `display` / `full` |
| `SMARTHIS_VENDOR_API_KEYS` | 空 | 厂商接口密钥，多个用英文逗号或分号分隔 |

厂商真实字段联调示例：

```powershell
$env:SMARTHIS_VENDOR_DATA_SCOPE="full"
$env:SMARTHIS_VENDOR_API_KEYS="vendor-secret"
npm start
```

访问 `dataScope=full` 或厂商数据库接口时，请携带：

```http
x-api-key: vendor-secret
```

或：

```http
Authorization: Bearer vendor-secret
```

## 9. 第三方系统对接

第三方对接由 `src/integrations.js` 提供角色化接口，当前支持：

- 第三方手术室门口屏。
- 第三方手术室信息显示与运行状态面板。
- 第三方排队叫号系统。

接口：

```http
GET /api/v1/integrations/profiles
GET /api/v1/integrations/or-door-screens/OR01/snapshot?date=2026-05-16
GET /api/v1/integrations/or-panels/OR01/snapshot?date=2026-05-16
GET /api/v1/integrations/queue-calls/snapshot?date=2026-05-16
GET /api/v1/integrations/queue-calls/snapshot?date=2026-05-16&dataScope=full
```

字段范围：

| `dataScope` | 说明 |
|---|---|
| `display` | 默认范围，按终端角色输出必要显示字段 |
| `full` | 授权联调范围，输出真实姓名、住院号、床号等角色必要字段 |

`full` 模式必须配合 `SMARTHIS_VENDOR_API_KEYS` 使用。

## 13. 当前边界

- 数据暂存在内存中，服务重启后恢复种子数据。
- 影像调阅是 DICOMweb 元数据、合成预览图和测试 DICOM 文件，不是真实 PACS。
- FHIR 和 HL7 当前是 MVP 级映射，后续需要按客户项目扩展字段、编码和消息段。
- 当前版本用于模拟、联调、测试和演示，不用于真实诊疗、真实收费、真实医保或真实病历归档。
- `D:\我的工作\AOV\客户资料\连云港中医院\11-17` 样本目录当前确认包含 DICOM、X 线检查报告 PDF、截图和压缩包，未发现可直接导入的 `.db/.sql/.mdb/.xlsx/.csv` 业务数据库文件；因此本项目只把该样本用于影像/报告/界面字段建模，不声称复刻真实医院数据库表结构。

## 14. 后续建议

1. 接 PostgreSQL，把内存状态迁移为持久化数据模型。
2. 增加 OpenAPI 文档和接口映射配置界面。
3. 接入 Orthanc、dcm4chee 或自研 PACS 模拟服务。
4. 增加 HL7 v2 入站监听、Webhook 订阅和接口重放配置。
5. 将控制台拆成独立前端，保留当前 Node 服务作为 API 和模拟引擎。
