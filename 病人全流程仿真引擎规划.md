# Patient Journey 病人全流程仿真引擎规划

版本：0.1  
日期：2026-05-16  
所属项目：SmartHIS 医院业务与接口仿真平台

## 1. 目标

Patient Journey 的目标是把 SmartHIS 从“接口模拟器”升级为“医院业务过程仿真平台”。

系统应支持一个虚拟病人从入院到出院的完整过程：

```text
建档/入院 -> 诊断/分床 -> 医嘱/检查/检验 -> 术前准备 -> 手术排班 -> 术中流程 -> 术后恢复 -> 出院归档
```

每一步既要改变 SmartHIS 内部数据，也要产生可观察的接口事件和时间线记录，用于首视 Smart 系列产品联调、演示、测试和客户培训。

## 2. 产品边界

### 包含

- 模拟医院业务状态流转。
- 模拟患者时间线、医嘱、报告、文书、手术、护理、费用和出院节点。
- 模拟 HIS、EMR、LIS、RIS、PACS、手麻、集成平台、首视产品接口事件。
- 支持场景回放、单步推进、一键跑完整流程。
- 支持多病种模板扩展。

### 不包含

- 不做真实临床诊断决策。
- 不作为真实医疗质控、真实医嘱审核或真实医保结算系统。
- 不导入真实患者隐私数据作为默认演示数据。
- 第一阶段不做完整药房、完整收费、完整 EMR 编辑器。

## 3. 第一阶段样板

第一阶段只做一个高价值样板：

**胆囊结石择期住院手术患者，从入院到出院。**

建议使用已有种子患者 `PAT000001 / ENC000001 / SCH000001` 作为默认演示旅程。

## 4. 标准旅程步骤

| 序号 | 步骤编码 | 步骤名称 | 主要数据变化 | 触发接口事件 |
| --- | --- | --- | --- | --- |
| 1 | `admission_registered` | 入院登记 | 创建/确认患者、住院、病区床位 | `journey.admission_registered` |
| 2 | `diagnosis_recorded` | 入院诊断 | 写入主诊断 | `journey.diagnosis_recorded` |
| 3 | `preop_orders_created` | 术前医嘱 | 检查检验和手术医嘱进入已开立 | `journey.preop_orders_created` |
| 4 | `reports_finalized` | 报告完成 | 检验/检查报告审核完成，影像可调阅 | `journey.reports_finalized` |
| 5 | `preop_summary_signed` | 术前小结签署 | EMR 文书进入已签署/Final | `journey.preop_summary_signed` |
| 6 | `anesthesia_visit_done` | 麻醉访视完成 | 生成麻醉访视文书 | `journey.anesthesia_visit_done` |
| 7 | `surgery_requested` | 手术申请确认 | 手术申请状态确认 | `journey.surgery_requested` |
| 8 | `surgery_scheduled` | 手术排班完成 | 排班、手术间、台次、人员确认 | `journey.surgery_scheduled` |
| 9 | `patient_called` | 接台 | 手术状态变为 Called | `surgery.called` |
| 10 | `patient_in_room` | 入室 | 手术状态变为 InRoom | `surgery.in_room` |
| 11 | `anesthesia_started` | 麻醉开始 | 手术状态变为 AnesthesiaStarted | `surgery.anesthesia_started` |
| 12 | `surgery_started` | 手术开始 | 手术状态变为 SurgeryStarted | `surgery.started` |
| 13 | `surgery_ended` | 手术结束 | 手术状态变为 SurgeryEnded | `surgery.ended` |
| 14 | `patient_out_room` | 出室 | 手术状态变为 OutRoom | `surgery.out_room` |
| 15 | `room_cleaned` | 清洁完成 | 手术间状态恢复 Idle | `or.cleaning_completed` |
| 16 | `postop_progress_recorded` | 术后病程 | 写入术后病程记录 | `journey.postop_progress_recorded` |
| 17 | `discharge_summary_signed` | 出院小结签署 | 写入出院小结 | `journey.discharge_summary_signed` |
| 18 | `patient_discharged` | 出院归档 | 住院结束、床位释放 | `encounter.discharged` |

## 5. 对首视产品的价值

| 首视模块 | Patient Journey 提供的价值 |
| --- | --- |
| 数字化手术室 | 可完整模拟术前资料、排班、术中状态、出室闭环 |
| 手术示教 | 可按旅程推进到直播开始，并调取患者脱敏摘要和影像 |
| 远程会诊 | 可在术前/术后节点生成会诊所需资料 |
| 家属谈话 | 可使用术前小结、诊断、手术计划等模拟谈话资料 |
| 门口屏 | 可显示接台、入室、手术开始、出室等真实状态 |
| 中央监控 | 可观察多个手术间与病人旅程状态 |
| 一键清洁 | 可由出室和清洁完成事件触发周转流程 |

## 6. 操作模式

### 单步推进

适合研发调试和培训：

```http
POST /api/v1/patient-journeys/{journeyId}/next
```

### 一键跑完整流程

适合售前演示和自动化测试：

```http
POST /api/v1/patient-journeys/{journeyId}/run
```

### 时间线查询

适合产品界面展示和接口排查：

```http
GET /api/v1/patient-journeys/{journeyId}/timeline
```

## 7. 扩展路线

### Phase 1

- 胆囊结石择期住院手术模板。
- 旅程模板、旅程实例、时间线、下一步、一键跑完。
- 控制台可查看和推进旅程。
- 旅程步骤写入接口日志。

### Phase 2

- 剖宫产流程。
- 骨科置换流程。
- 急诊手术流程。
- 日间手术流程。
- 远程会诊专项流程。
- 示教直播专项流程。

### Phase 3

- 旅程模板编辑器。
- 虚拟时钟。
- 旅程参数化：病种、科室、床位、手术间、医生、时间间隔。
- 自动化测试用例导出。
- 客户医院模板包。

## 8. 第一阶段验收标准

- 可以查询旅程模板和当前病人旅程。
- 可以创建一个胆囊结石住院手术旅程。
- 可以单步推进并看到病人时间线增长。
- 推进到手术步骤时，会同步改变现有手术排班状态。
- 推进到出院时，会释放床位、结束住院、产生出院小结。
- 控制台能显示病人旅程进度和时间线。
- 每个旅程步骤都写入接口日志，便于首视产品联调排查。
