# 手术室门口屏对接场景设计说明

版本：0.1  
日期：2026-05-20  
范围：SmartHIS 模拟手术室内安卓终端、手术室门口屏、家属等待区、护士站、院领导看板之间的手术状态联动

## 1. 结论

本场景不应设计为“室内安卓终端直接推送门口屏”。推荐采用：

```text
室内安卓终端 -> SmartHIS 手术状态服务 -> 状态事件分发 -> 各类展示终端
```

服务端是唯一事实源。室内终端只提交状态变更请求；SmartHIS 校验终端、人员、房间、状态流转后生成状态事件；门口屏、家属区、护士站、院领导看板只消费经过服务端投影后的展示数据。

## 2. 现有基础

SmartHIS 当前已经具备以下基础能力：

- 手术排班模型：`surgerySchedules`。
- 手术状态事件：`surgeryEvents`。
- 状态更新接口：`PATCH /api/v1/surgery-schedules/{scheduleId}/status`。
- 当前手术间查询：`GET /api/v1/operating-rooms/{roomId}/current-surgery`。
- 家属等待区大屏：`GET /family-waiting-display`。
- 手术室门口屏：`GET /or-door-display?roomId=OR01`。
- 手术部护士站看板：`GET /or-nurse-station-display`。
- 院长质控看板：`GET /director-quality-display`。

下一步应在这些能力上补齐终端身份、权限控制、状态流转约束、分角色展示投影、断线恢复和审计，而不是另建一套孤立屏幕系统。

当前服务端 MVP 已补充：

- `GET /api/v1/or-terminals`、`POST /api/v1/or-terminals/register`、`POST /api/v1/or-terminals/{deviceId}/heartbeat` 等终端接口。
- `GET /api/v1/or-display/rooms/{roomId}/door-snapshot` 等分角色展示快照接口。
- 带 `deviceId` 的手术状态更新会校验终端写权限、房间绑定、操作人员、状态流转和幂等键。
- 家属等待区快照使用独立脱敏投影，不复用完整手术排班对象。

当前页面 MVP 已补充：

- `GET /or-inner-control?roomId=OR01`：模拟手术室内安卓控制终端。
- `GET /or-terminal-simulator`：在单台电脑的一个页面内同时模拟室内控制端、门口屏、家属区、护士站、院领导看板和终端心跳状态，适合没有真实安卓终端时做联调验证。
- 该模拟台只用于测试展示和联调，不改变后端权限模型；状态更新仍必须通过具体 `deviceId`、`operatorId` 和手术间绑定校验。

## 3. 场景边界

### 3.1 本期目标

- 模拟每间手术室配置 2 台安卓终端：室内控制终端、门口展示终端。
- 室内控制终端可以提交本房间手术状态变更。
- 门口屏、家属区、护士站、院领导看板同步更新状态。
- 不同终端根据角色展示不同字段。
- 所有状态变更可追溯、可回放、可审计。
- 支持终端离线、断线恢复、重复点击和非法状态跳转的模拟。

### 3.2 非目标

- 不直接对接真实医院生产 HIS、EMR、手麻系统。
- 不处理真实患者个人信息。
- 不做真实诊疗、收费、医保或病历归档。
- 不在第一期开发原生安卓 App；先用 Web 页面模拟安卓终端，稳定后再封装 WebView。

## 4. 终端与权限模型

### 4.1 终端类型

| 终端类型 | 示例位置 | 写权限 | 数据范围 | 说明 |
| --- | --- | --- | --- | --- |
| 室内控制终端 | OR01 手术室内 | 有 | 本手术间 | 控制手术状态，必须记录操作人员 |
| 门口展示终端 | OR01 门口 | 无 | 本手术间 | 只读展示当前手术状态 |
| 家属等待区屏 | 家属区 | 无 | 今日手术脱敏队列 | 只显示脱敏状态，不显示诊断和详细术式 |
| 护士站看板 | 手术部护士站 | 有限 | 手术部全部房间 | 可查看全量状态，可人工纠偏 |
| 院领导看板 | 管理端 | 无 | 汇总指标 | 默认展示运营与质控指标，不展示患者明细 |
| 管理端 | 信息科/手术部管理员 | 有 | 全部 | 终端绑定、停用、权限配置 |

### 4.2 双身份要求

终端控制权不能只依赖安卓设备 ID。状态变更必须同时具备：

- 设备身份：`deviceId`，绑定终端类型、房间、物理位置。
- 人员身份：`operatorId`，绑定医生、护士、麻醉医生、管理员等角色。

第一期可以用请求头或请求体模拟身份；后续再扩展为扫码登录、工牌登录、统一认证或设备证书。

### 4.3 权限规则

| 操作 | 室内控制终端 | 门口屏 | 家属区屏 | 护士站 | 院领导看板 | 管理端 |
| --- | --- | --- | --- | --- | --- | --- |
| 查看本房间状态 | 允许 | 允许 | 不适用 | 允许 | 汇总 | 允许 |
| 查看全部房间状态 | 不允许 | 不允许 | 脱敏 | 允许 | 汇总 | 允许 |
| 提交状态变更 | 允许，仅本房间 | 不允许 | 不允许 | 允许，需原因 | 不允许 | 允许，需原因 |
| 绑定终端 | 不允许 | 不允许 | 不允许 | 不允许 | 不允许 | 允许 |
| 查看审计日志 | 不允许 | 不允许 | 不允许 | 允许 | 汇总 | 允许 |

## 5. 状态字典与流转

### 5.1 状态字典

内部代码可以继续沿用现有 `SURGERY_STATUS`；对外展示必须使用中文。

| 内部状态 | 业务展示 | 家属区展示 | 说明 |
| --- | --- | --- | --- |
| `Scheduled` | 已排班 | 等待中 | 已完成手术排班 |
| `Called` | 已接台 | 等待接入 | 手术部已呼叫或接台 |
| `InRoom` | 已入室 | 已进入手术区 | 患者进入手术间 |
| `AnesthesiaStarted` | 麻醉开始 | 手术准备中 | 麻醉开始 |
| `SurgeryStarted` | 手术开始 / 手术中 | 手术中 | 手术正式开始 |
| `SurgeryEnded` | 手术结束 | 手术已结束 | 主体手术结束 |
| `OutRoom` | 已出室 | 复苏观察中 / 已离开手术区 | 患者离开手术间 |
| `Cleaning` | 清洁中 | 不展示 | 手术间周转清洁 |
| `Completed` | 已完成 / 可接台 | 不展示 | 房间周转完成 |
| `Cancelled` | 已取消 | 请咨询护士站 | 手术取消或改期 |

`可接台` 不建议第一期新增为手术状态。它更适合作为房间状态，由 `Completed` 加房间空闲状态推导，避免把“患者手术流程”和“房间周转能力”混在同一个状态字段中。

### 5.2 默认流转

```text
已排班 -> 已接台 -> 已入室 -> 麻醉开始 -> 手术开始
-> 手术结束 -> 已出室 -> 清洁中 -> 已完成/可接台
```

允许从 `已排班`、`已接台` 进入 `已取消`。  
护士站或管理端允许人工纠偏，但必须填写原因。  
急诊插台、抢救等越级状态变更允许模拟，但必须记录 `overrideReason`。

### 5.3 状态变更校验

- 室内控制终端只能控制绑定手术间。
- 门口屏和家属区屏没有状态写权限。
- 已完成或已取消的手术不能继续推进，除非管理员纠偏。
- 重复提交同一个状态需要幂等处理，不能重复生成有效业务事件。
- 多终端并发操作时使用 `eventVersion` 或 `lastEventId` 做冲突校验。
- 任何跳过前置状态的操作必须记录原因。

## 6. 数据模型扩展

### 6.1 TerminalDevice 终端设备

| 字段 | 说明 |
| --- | --- |
| deviceId | 终端 ID |
| deviceCode | 设备编码 |
| deviceName | 设备名称 |
| terminalType | 室内控制终端、门口展示终端、家属区屏、护士站、院领导看板 |
| roomId | 绑定手术间，家属区和院领导看板可为空 |
| locationName | 物理位置 |
| permissionProfile | 权限模板 |
| status | 在线、离线、停用、维修 |
| lastHeartbeatTime | 最近心跳时间 |
| registeredTime | 注册时间 |
| enabled | 是否启用 |

### 6.2 SurgeryStatusEvent 手术状态事件

现有 `surgeryEvents` 可以扩展为以下结构：

| 字段 | 说明 |
| --- | --- |
| eventId | 事件 ID |
| surgeryScheduleId | 手术排班 ID |
| roomId | 手术间 |
| previousStatus | 原状态 |
| newStatus | 新状态 |
| eventType | 事件类型 |
| eventTime | 事件时间 |
| operatorId | 操作人员 |
| deviceId | 操作设备 |
| sourceSystem | 来源系统 |
| eventVersion | 事件版本 |
| idempotencyKey | 幂等键 |
| overrideReason | 越级或纠偏原因 |
| payload | 扩展数据 |

### 6.3 DisplayProjection 展示投影

不同屏幕不直接读取完整手术对象，而读取各自的投影视图：

| 投影 | 用途 |
| --- | --- |
| `doorDisplaySnapshot` | 门口屏本房间展示 |
| `familyDisplaySnapshot` | 家属区脱敏队列 |
| `nurseStationSnapshot` | 护士站全局调度 |
| `directorDashboardSnapshot` | 院领导汇总指标 |

## 7. 展示脱敏规则

### 7.1 门口屏

门口屏处于手术部区域，但仍应按医院配置控制展示粒度。

默认展示：

- 手术间名称。
- 当前状态。
- 计划时间、当前时间、最后更新时间。
- 患者姓名脱敏显示，例如 `张*`。
- 性别、年龄。
- 手术名称可配置：默认显示，敏感专科可隐藏或显示为“择期手术”。
- 主刀、麻醉医生、巡回护士可配置显示。

不默认展示：

- 身份证号。
- 完整住院号。
- 详细诊断。
- 费用、医保、联系电话。

### 7.2 家属等待区

默认只显示：

- 排队序号或手术通知号。
- 手术间或等候区域。
- 患者姓名脱敏。
- 状态：等待中、已进入手术区、手术准备中、手术中、手术已结束、复苏观察中、请咨询护士站。
- 最后更新时间。

不显示：

- 诊断。
- 详细术式。
- 医生姓名。
- 住院号、身份证、手机号。
- 异常原因明细。

### 7.3 护士站

护士站可以展示业务调度所需明细：

- 全部手术间状态。
- 患者、术式、科室、主刀、麻醉、护士。
- 延误、取消、清洁周转、终端离线。
- 状态事件历史。
- 人工纠偏入口。

### 7.4 院领导看板

默认展示汇总指标：

- 今日手术台次。
- 进行中台次。
- 已完成台次。
- 延误台次。
- 平均接台时间。
- 房间利用率。
- 清洁周转时长。
- 异常和终端离线数量。

不默认展示患者明细，除非进入授权的质控追溯模式。

## 8. 接口设计

### 8.1 终端注册与心跳

```http
POST /api/v1/or-terminals/register
PATCH /api/v1/or-terminals/{deviceId}/binding
PATCH /api/v1/or-terminals/{deviceId}/status
POST /api/v1/or-terminals/{deviceId}/heartbeat
GET /api/v1/or-terminals
GET /api/v1/or-terminals/{deviceId}
```

### 8.2 状态控制

第一期可复用现有接口，并增加设备和人员字段：

```http
PATCH /api/v1/surgery-schedules/{scheduleId}/status
Content-Type: application/json

{
  "status": "手术开始",
  "operatorId": "PRA001",
  "deviceId": "DEV_OR01_INNER",
  "sourceSystem": "室内控制终端",
  "idempotencyKey": "DEV_OR01_INNER-SCH000001-SurgeryStarted-202605200930",
  "lastEventId": "EVT000003",
  "overrideReason": ""
}
```

后续可以增加面向终端的语义接口：

```http
POST /api/v1/or-control/status-events
```

该接口由服务端根据 `deviceId` 自动识别房间和可控制手术，减少终端传错 `scheduleId` 的风险。

### 8.3 展示快照

```http
GET /api/v1/or-display/rooms/{roomId}/door-snapshot
GET /api/v1/or-display/family-waiting-snapshot?date=2026-05-20
GET /api/v1/or-display/nurse-station-snapshot?date=2026-05-20
GET /api/v1/or-display/director-dashboard-snapshot?date=2026-05-20
```

### 8.4 实时分发

第一期可以使用轮询，降低实现复杂度：

```text
门口屏：3 秒轮询本房间快照
家属区：5 秒轮询脱敏队列
护士站：3 秒轮询全局快照
院领导看板：15 秒轮询汇总指标
```

第二期再引入：

```http
GET /api/v1/or-events/stream
```

采用 SSE 或 WebSocket。若后续要模拟大量安卓终端，再评估 MQTT。

## 9. 异常场景

| 场景 | 系统行为 |
| --- | --- |
| 室内终端离线 | 护士站显示离线，门口屏继续显示最近快照和最后更新时间 |
| 门口屏离线 | 护士站显示门口屏离线，不影响室内控制 |
| 家属区屏断线 | 重连后重新拉取脱敏快照 |
| 重复点击状态按钮 | 返回已接受或幂等结果，不重复生成有效事件 |
| 非法跳转 | 拒绝并返回可允许的下一状态 |
| 急诊越级 | 允许授权人员操作，必须填写原因 |
| 手术取消 | 门口屏显示已取消，家属区显示请咨询护士站 |
| 手术间临时停用 | 护士站和管理端显示停用，门口屏显示暂停使用 |

## 10. 测试与验收

### 10.1 自动化测试

至少覆盖：

- 室内终端能更新本房间状态。
- 室内终端不能更新其他房间状态。
- 门口屏不能提交状态变更。
- 家属区快照不包含诊断、身份证、手机号、完整住院号。
- 状态非法跳转被拒绝。
- 重复提交同一幂等键不会生成重复有效事件。
- 设备心跳超时后护士站显示离线。
- 状态变更后各展示快照同步变化。

### 10.2 演示验收

建议使用 3 间手术室：

- OR01：正在手术。
- OR02：等待接台。
- OR03：清洁中。

演示链路：

```text
室内控制终端点击“患者入室”
-> 门口屏显示“已入室”
-> 家属区显示“已进入手术区”
-> 护士站 OR02 状态同步变化
-> 院领导看板进行中/等待中指标更新
-> 审计日志记录设备、人员、时间和状态
```

验收标准：

- 状态同步延迟在模拟环境下不超过 3 秒。
- 所有外部展示使用中文状态。
- 家属区无敏感明细泄露。
- 每次状态变更可回放。
- 断线恢复后屏幕显示服务端最新快照。

## 11. 开发顺序

### 第一步：模型和文档

- 补充 `TerminalDevice`、`DeviceHeartbeat`、`DisplayProjection` 设计。
- 明确状态流转和权限矩阵。
- 保持当前 `surgerySchedules`、`surgeryEvents` 为主线。

### 第二步：服务端 MVP

- 新增终端设备种子数据。
- 新增终端注册、绑定、心跳接口。
- 扩展状态更新接口，校验 `deviceId`、`operatorId`、房间绑定和幂等键。
- 新增展示快照 API。

### 第三步：页面 MVP

- 增加室内控制终端页面：`/or-inner-control?roomId=OR01`。
- 门口屏、家属区、护士站、院领导看板改为读取展示快照。
- 护士站显示终端在线状态和最近更新时间。

### 第四步：异常和审计

- 增加非法跳转拦截。
- 增加人工纠偏原因。
- 增加事件回放接口。
- 增加测试覆盖。

### 第五步：安卓封装

- 先用安卓浏览器或 WebView 全屏模式验证。
- 稳定后再做原生壳：开机自启、锁定退出、设备注册、远程配置。
- 真实项目再评估 MDM、证书、专网、网闸、日志留存和等级保护要求。

## 12. 风险与控制

| 风险 | 控制措施 |
| --- | --- |
| 状态被误操作 | 双身份、状态流转校验、人工纠偏原因 |
| 家属区隐私泄露 | 独立脱敏投影，不直接复用完整排班对象 |
| 终端丢失或被挪用 | 设备绑定、后台停用、心跳监控 |
| 屏幕显示旧状态 | 最后更新时间、断线提示、重连拉取快照 |
| 多端并发冲突 | 事件版本、幂等键、服务端事实源 |
| 后续接真实系统困难 | 保留 HIS/手麻/集成平台事件边界，不让终端直连第三方系统 |

## 13. 外部依据

- 国家卫生健康委《全国医院信息化建设标准与规范（试行）》：覆盖业务应用、信息平台、基础设施、安全防护和终端安全等建设要求。
- [GB/T 22239-2019 信息安全技术 网络安全等级保护基本要求](https://openstd.samr.gov.cn/bzgk/std/newGbInfo?hcno=BAFB47E8874764186BDB7865E8344DAF)：用于约束终端、网络、访问控制和审计。
- [中华人民共和国个人信息保护法](https://www.cac.gov.cn/2021-08/20/c_1631050028355286.htm)：医疗健康信息属于敏感个人信息，家属区和公共展示必须遵循最小必要原则。
